SubViewManager

new SubViewManager (subViewCfg, parent, options)

Local constructor intended for use by Backbone.BaseView. Creates an object specifically for manipulating subviews. Is created automatically by the BaseView instance and accessed through the '.subs' property of the the instance.

Parameters:
Name Type Argument Description
subViewCfg object

An object with a configuration to add for subviews (see addConfig(map))
parent Backbone.BaseView

The parent BaseView instance
options <optional>

addConfig (name, construct, options, location, singleton) → {SubViewManager}

Add an configuration for a subview to the SubViewManager. It can be instantiated later with the add function.

Parameters:
Name Type Argument Description
name String

The name or type that you would like use to refer to subViews created with this config
construct String | Function

The constructor function for this subView. If its a string, the string must resolve to a global name-spaced function in dot notation. (e.g. 'Backbone.imageUpload.View')
options String <optional>

The options or base options you would like to pass to the constructor whenever it is initialized.
location String | HTMLElement <optional>

If you want to be able to automatically place a subView's DOM element somewhere in the parent view, pass a selector or DOM element or $ instance here that can be used by jquery as a wrapper to append the subView $el
singleton Boolean <optional>

If you want this view to be a singleton subView. A singleton subView will only allow one instance of it to be created.

Render a subView specified by a key (from their configuration key). The key will retrieve subViews by looking for a type and then looking for singletons or specific instance using the Model/View cid

A key, type, View/Model cid, Model, or View to refer to a specific subView or subViews. For example, passing the view cid 'view4' will render only the view with that cid, and passing the type 'creatorNames' will render views matching the type 'creatorNames'.

Partial applications of underscore (or lodash if you use that instead) functions of the same name (operates on the subViews array).
